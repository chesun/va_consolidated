### Bug 2: Header-vs-code OUTPUTS mismatch (M1 -3 from coder-critic)

Initial draft of `va_sib_lag_spec_fb_tab.do` header claimed:
```
$tables_dir/va_cfr_all_v[12]/combined/sib_lag_fb_spec_<outcome>.csv
```

Coder-critic grepped the body and found the actual writes go to:
```
$tables_dir/va_cfr_all_v[12]/{spec_test,fb_test}/{spec,fb}_sib_lag.dta
```

Pure **derive-don't-guess violation**: I assumed CSV-output filename pattern from sister `va_spec_fb_tab.do` (batch 3b) without grepping the actual body of this file. The body uses `regsave using ... .dta`, not `esttab using ... .csv`.

**Fixed in-commit before push** by:
1. Running `grep -nE "regsave using|save," <file>` to identify actual write paths
2. Rewriting the header OUTPUTS section to match
3. Flagging `combined/` mkdir prep as dead code (verbatim-preserved per ADR-0021)

### Lesson recurrence — codified

This is the 3rd time the grep-before-claim discipline has been needed (batches 3b, 3c2, 3d). Each time, verbatim-preservation made me feel "safe" copying headers from prior files. **Process change for batch 4+: grep the OUTPUTS paths from the file body BEFORE writing the header, not after.**

Added to TODO.md pre-batch checklist as item 5.
