## Key findings

### Finding 1: Critical bug in batch 2b (post-`cd $vaprojdir` relative includes)

While preparing the LEGACY repointing edits for create_score_samples.do, I noticed `include do/va/helpers/macros_va.doh` at L188. Tracing through: predecessor pattern `cd $vaprojdir` at L169 sets CWD to `$vaprojdir`, so `include do/...` at L188 would resolve to `$vaprojdir/do/...` — a path that doesn't exist in the consolidated layout. **9 sites total** across 3 batch 2b files have this bug:

- `touse_va.do`: 3 sites
- `create_score_samples.do`: 3 sites
- `create_out_samples.do`: 3 sites

This would have caused runtime FileNotFoundError when `do_touse_va = 1` or `do_create_samples = 1` were flipped on. Coder-critic round-1 on batch 2b missed it (focused on sandbox-write + LEGACY-include + DEAD INCLUDE concerns; didn't simulate post-cd relative resolution).

**Decision:** Bundle the fix with batch 2c — it's logically coherent ("wire in merge helpers + fix the dependent bug introduced in same parent files"). Convention going forward: any consolidated `include`/`do` in a script with `cd $vaprojdir` must use absolute `$consolidated_dir/do/...` prefix. Codified in TODO.md "Pre-batch checklist" item 3 for Step 3+.

### Finding 2: macros_va.doh consumers in merge_va_smp_acs.doh

Pre-relocation merge_va_smp_acs.doh L47 includes `$vaprojdir/do_files/sbac/macros_va.doh` to set up its required locals (`census_grade`, `census_controls`, ``va_type'_min_year`, ``va_type'_max_year`). These are used at body-positions L62, L100. Relocated form: `include $consolidated_dir/do/va/helpers/macros_va.doh`. Coder-critic verified all 4 locals are present in the consolidated macros_va.doh (L114-119, L138-143, L239, L243-248).

### Finding 3: Pure-fragment vs do-script distinction matters for header structure

Three of the four helpers (`merge_loscore`, `merge_sib`, `merge_lag2_ela`) are pure parent-context fragments — they `include` into the parent's scope, no own log/save, rely on parent locals. The fourth (`merge_va_smp_acs`) is called via `do` with positional args, so it has its own scope and needs its own `include` of macros_va.doh. Headers explicitly distinguish to prevent future-reader confusion.
